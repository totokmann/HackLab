XSS

Dificultad: Facil

Descripción: XSS (Cross-Site Scripting) es un tipo de vulnerabilidad de seguridad en aplicaciones web que permite a un atacante inyectar código malicioso, generalmente en forma de scripts JavaScript, en páginas web vistas por otros usuarios. Esto puede ocurrir cuando la aplicación web no valida o sanitiza adecuadamente los datos de entrada proporcionados por los usuarios.

    Hay tres tipos principales de XSS:
  • Reflejada (Non-Persistent): El código malicioso se inyecta a través de una solicitud HTTP y se refleja en la respuesta del servidor sin ser almacenado. Por ejemplo, en un enlace malicioso enviado por correo electrónico.
  • Almacenada (Persistent): El código malicioso se almacena en el servidor (por ejemplo, en una base de datos) y se ejecuta cada vez que un usuario accede a la página afectada.
  • DOM-Based: El código malicioso se inyecta y ejecuta directamente en el navegador del usuario, manipulando el Document Object Model (DOM) de la página web.

    • Las consecuencias de una vulnerabilidad XSS pueden incluir el robo de cookies de sesión, la falsificación de solicitudes, o el secuestro de cuentas de usuario.
