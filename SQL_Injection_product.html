SQL Injection

Dificultad: Facil

Una vulnerabilidad de SQL Injection (Inyección de SQL) ocurre cuando una aplicación web permite que un atacante interfiera con las consultas SQL que realiza la aplicación a su base de datos. Esto sucede cuando los datos de entrada del usuario no se validan o escapan correctamente, lo que permite al atacante manipular las consultas SQL originales.

    Tipos comunes de SQL Injection:
  • Basada en consultas: El atacante modifica una consulta SQL existente. Por ejemplo, alterando una consulta de autenticación para siempre devolver un resultado positivo.
  • Basada en union: El atacante utiliza la cláusula UNION para combinar resultados de varias consultas, obteniendo acceso a datos que normalmente no serían visibles.
  • Basada en ciegas: Ocurre cuando el servidor no muestra los resultados de la consulta al atacante. El atacante realiza consultas SQL que modifican el comportamiento del servidor, permitiendo inferir información de la base de datos indirectamente.

    Consecuencias
  • Acceso no autorizado a datos sensibles (nombres de usuario, contraseñas, información personal).
  • Modificación o eliminación de datos en la base de datos.
  • Compromiso total del servidor, dependiendo de los permisos de la cuenta de la base de datos utilizada por la aplicación.
